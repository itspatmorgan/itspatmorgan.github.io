## How we work

This site is built collaboratively with [Claude Code](https://claude.ai/code). Our workflow:

1. **Plan** — Implementation plans are created in [`.claude/plans/`](https://github.com/itspatmorgan/itspatmorgan.github.io/tree/main/.claude/plans) before writing code
2. **Track** — Work is tracked as GitHub Issues on the [project board](https://github.com/users/itspatmorgan/projects/2), with plans linked from issue comments
3. **Build** — Code is written, reviewed, and committed with descriptive messages
4. **Document** — Decisions and context are preserved in issues and plans for future reference

Patrick provides input context (briefs, outlines, reference assets) in [`.reference/`](https://github.com/itspatmorgan/itspatmorgan.github.io/tree/main/.reference). Claude Code produces implementation plans in `.claude/plans/`.

## Labels

Issues are tagged across two dimensions — **where** the work happens and **what** kind of work it is:

| Where | What |
|-------|------|
| `home` — Home page | `design` — Layout, styling, typography, branding, animations |
| `resume` — Resume page | `content` — Articles, case studies, media assets |
| `projects` — Project detail pages | `infrastructure` — Build, deploy, tooling, dev workflow |
| `writing` — Writing section | `documentation` — Docs, guides, READMEs |
